## 0. 이번 수행평가의 핵심

이번 수행평가는 파이썬으로 **나만의 자유 주제 프로그램**을 만드는 프로젝트입니다.

단순히 코드가 실행되는지만 평가하지 않습니다. 다음을 함께 봅니다.

## 4. 2차원 리스트란?

2차원 리스트는 **리스트 안에 리스트가 들어 있는 구조**입니다.

예를 들어 음식 추천기라면 다음처럼 쓸 수 있습니다.

```python
foods = [
    ["김밥", 3500, "식사", "가벼움"],
    ["라면", 4000, "식사", "매움"],
    ["아이스티", 2500, "음료", "시원함"],
]
```

이때 한 행은 음식 하나를 의미합니다.

| 열 번호 | 의미 |
|---:|---|
| 0번 열 | 이름 |
| 1번 열 | 가격 |
| 2번 열 | 종류 |
| 3번 열 | 특징 |

예를 들어:

```python
print(foods[0][0])  # 김밥
print(foods[0][1])  # 3500
print(foods[2][3])  # 시원함
```

코드인터뷰에서 반드시 이런 질문을 받을 수 있습니다.

```text
네 2차원 리스트에서 한 행은 무엇을 의미하니?
각 열은 무엇을 의미하니?
```

## 5. 함수 분리란?

함수 분리는 큰 프로그램을 작은 기능으로 나누는 것입니다.

예를 들어 추천 프로그램이라면 다음처럼 나눌 수 있습니다.

```python
def get_user_input():
    # 사용자 입력 받기
    pass


def find_items(data, user_condition):
    # 조건에 맞는 항목 찾기
    pass


def print_result(results):
    # 결과 출력하기
    pass
```

좋은 함수 분리는 다음과 같습니다.

```text
- 함수 이름만 봐도 역할이 보인다.
- 한 함수가 너무 많은 일을 하지 않는다.
- 입력, 처리, 출력 기능이 적절히 나뉘어 있다.
- main()에서 전체 실행 흐름을 확인할 수 있다.
```

## 6. GitHub Codespaces에서 작업하기

### 6-1. 내 저장소 만들기

선생님이 공유한 템플릿 저장소에서 다음 순서로 내 저장소를 만듭니다.

저장소 이름은 선생님 안내에 맞춰 작성합니다.

내 저장소에서 다음 순서로 실행합니다.

```text
Code
```

## 7. 실행 방법

터미널에서 다음 명령을 실행합니다.

```bash
python main.py
```

실행 오류가 나면 먼저 다음을 확인합니다.

```text
1. 파일 이름이 main.py가 맞는가?
2. 들여쓰기가 맞는가?
3. 괄호, 따옴표, 콜론(:)이 빠지지 않았는가?
4. input()으로 받은 값을 숫자로 계산할 때 int()를 사용했는가?
5. 리스트 인덱스 번호가 범위를 벗어나지 않았는가?
```

## 8. 매 차시 끝에 반드시 저장하기

Codespaces에서 파일을 수정한 뒤에는 **저장 + Commit + Sync**를 해야 GitHub 저장소에 반영됩니다.

매 시간 끝나기 전에 다음을 합니다.

```text
1. Ctrl + S로 저장
2. 왼쪽 Source Control 아이콘 클릭
3. 메시지 입력 예: 1차시 작업 완료
4. Commit 클릭
5. Sync Changes 클릭
6. Codespace Stop
```

터미널을 사용할 수 있다면 다음 명령으로 해도 됩니다.

```bash
git add .
git commit -m "class progress"
git push
```

브라우저 창만 닫으면 Codespace가 계속 켜져 있을 수 있습니다. 수업이 끝나면 반드시 Codespace를 Stop 하세요.

## 9. 학교 계정 Gemini 사용 규칙

이번 수행평가에서는 **학교 계정 Gemini**를 공식 AI 도구로 사용합니다.

Gemini는 다음 용도로 사용할 수 있습니다.

```text
- 주제 아이디어 얻기
- 입력-처리-출력 구조 정리하기
- 2차원 리스트 구조 아이디어 얻기
- 함수 분리 방법 질문하기
- 오류 원인 찾기
- 테스트 사례 만들기
- 보고서 작성 방향 점검하기
```

하지만 다음은 금지합니다.

```text
- Gemini가 만든 코드를 이해하지 못한 채 그대로 제출하기
- ai_log.md에 AI 활용 기록을 남기지 않기
- 실제 개인정보, 친구 이름, 성적, 전화번호, 주소 등을 Gemini에 입력하기
- 본인이 설명할 수 없는 복잡한 코드를 제출하기
- 수업에서 배우지 않은 외부 라이브러리, 웹앱, GUI 등을 허락 없이 사용하기
```

프롬프트 예시는 [`docs/gemini_prompt_bank.md`](docs/gemini_prompt_bank.md)를 보세요.

## 12. 높은 점수를 받으려면

기본 조건을 만족하면 중간 점수까지 받을 수 있습니다. 높은 점수를 받으려면 다음이 필요합니다.

```text
1. 주제가 구체적이고 문제 상황이 분명하다.
2. 입력-처리-출력 구조가 명확하다.
3. 2차원 리스트의 행과 열의 의미가 분명하다.
4. 조건문과 반복문이 단순 출력이 아니라 실제 판단, 검색, 계산, 추천, 판정 등에 사용된다.
5. 함수가 기능별로 나뉘어 있다.
6. 사용자 입력이나 조건에 따라 결과가 달라진다.
7. 결과가 없거나 잘못된 입력이 들어오는 경우를 고려했다.
8. Gemini가 제안한 내용을 그대로 사용하지 않고 자신의 주제에 맞게 수정했다.
9. 실행 결과와 개선 방안을 보고서에 구체적으로 작성했다.
10. 코드인터뷰에서 자기 코드의 핵심 부분을 설명할 수 있다.
```
